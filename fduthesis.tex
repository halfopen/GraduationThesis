\begin{abstract}
  % 本文通过调研人脸识别技术应用到面诊进行日常的健康管理，探索在日常健康管理的场景下会遇到的问题并提出了交互上改进的方法。
  % 根据调研的结果和发现，以云中医app为原型，重新设计了整个中医面诊系统，提高系统的可用性和交互性。
  % 同时，针对面诊系统的可解释性的问题，重新实现诊断算法，在用户使用的过程中加入算法透明性，经过用户调研进行对比实验，探索了面诊算法的透明性在中医诊断系统中的作用。

% 本文对日常健康管理场景下的健康管理技术进行了定性研究，探索出了一系列日常使用场景下，基于面部的健康管理技术可能会遇到的问题，并提出了改进方法和设计原则。
% 根据定性研究发现，当前还没有一款比较好地能够支持对健康诊断技术做交互研究的平台。通过对交互研究实验进行抽象建模，本文设计并实现了平台的以下功能：

% 1. 模型管理

% 2.任务分配与调度

% 3.日志管理

随着移动设备硬件计算能力的提高已经中医面诊信息化技术的进步，在移动设备上进行日常面诊已经成为可能。
目前已有的面诊信息化技术主要是考虑在诊所环境下使用，而人机交互领域的日常健康管理技术主要集中在健康行为劝导和慢性病管理。
在这个背景下，本文探索了日常环境下，面诊交互技术应该如何设计的问题。

本文的主要工作如下：

（1）利用技术探针进行日常场景下面诊应用的定性研究，对参与者进行深度访谈，将转录后的录音进行文字编码提取，探索出了日常场景下面诊应用会遇到的问题并提出了设计方面的原则。

（2）在开展交互实验之前，对人机交互实验的流程进行建模，设计并实现了用于快速开展面诊应用相关实验的实验平台：实现了系统高可用，快速迭代，模型任务模块分离，日志管理，问卷关联等功能，
提高了系统的稳定性，后面大规模、快速地开展新的实验。

（3）基于定性研究中得出的设计原则，在实验平台上对透明性的具体实现方案进行了实验。

\end{abstract}
