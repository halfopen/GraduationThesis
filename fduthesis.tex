\begin{abstract}
  % 本文通过调研人脸识别技术应用到面诊进行日常的健康管理，探索在日常健康管理的场景下会遇到的问题并提出了交互上改进的方法。
  % 根据调研的结果和发现，以云中医app为原型，重新设计了整个中医面诊系统，提高系统的可用性和交互性。
  % 同时，针对面诊系统的可解释性的问题，重新实现诊断算法，在用户使用的过程中加入算法透明性，经过用户调研进行对比实验，探索了面诊算法的透明性在中医诊断系统中的作用。

% 本文对日常健康管理场景下的健康管理技术进行了定性研究，探索出了一系列日常使用场景下，基于面部的健康管理技术可能会遇到的问题，并提出了改进方法和设计原则。
% 根据定性研究发现，当前还没有一款比较好地能够支持对健康诊断技术做交互研究的平台。通过对交互研究实验进行抽象建模，本文设计并实现了平台的以下功能：

% 1. 模型管理

% 2.任务分配与调度

% 3.日志管理



随着目前人们生活节奏的加快以及养生意识的提高，越来越多的人越来越关注自己的健康问题，但是紧凑的工作时间让大多数人无法单独抽出时间去医院定期检查自己的健康状况。
随着移动设备的普及已经更新，手机硬件计算能力得到大大地提高，越来越多的实际应用可以在日常场景下完成，大量关于日常场景下的健康追踪技术也在不断深入。
同时，近几年人工智能技术的快速发展，健康诊断算法的能力也在不断地提高，特别是基于面部信息诊断技术也在不断突破，在移动设备上基于面部信息进行日常健康诊断已经成为可能。

目前已有的面诊信息化技术主要是考虑在诊所环境下使用，而人机交互领域的日常健康管理技术主要集中在用户健康行为引导和如何帮助用户更好地进行慢性病管理等方面。
在这个背景下，本文对日常环境下的面诊交互技术应该如何设计的问题进行了探索，提出了对应的设计规则并通过实验平台进行了验证。
本文的主要工作如下：

(1) 利用技术探针进行日常场景下面诊应用的定性研究，对参与者进行深度访谈，将转录后的录音进行文字编码提取，探索出了日常场景下面诊应用会遇到的问题并提出了设计方面的原则：增加可变性、系统透明性和日常可用性。

(2) 设计并实现了可拓展的实验平台。在开展交互实验之前，为了提高后续实验的效率，本文对人机交互实验的流程进行建模，设计并实现了用于快速开展面诊应用相关实验的实验平台：实现了系统高可用，快速迭代，模型任务模块分离，日志管理，问卷关联等功能，提高了系统的稳定性，能够应对后面大规模的用户实验场景，并且快速地开展新的实验。

% 通过调用实验平台接口对跨平台客户端进行实现并通过实验验证设计规则的有效性。模型运行在服务器，解决了用户设备的差异性带来的稳定性和性能的问题。

(3) 为了验证设计规则，本文分别实现了解决可用性问题的原型和添加解释的原型，并通过实地调研的方式验证了设计规则的有效性。
实验的结果和分析表明，本文提出的提高可用性的设计规则的确能够提高系统使用的便利性，能够方便用户在日常健康的场景下使用；而本文提出的增加系统透明性的设计原则，的确能够在目前算法的结果不完美的情况下，提高用户对系统的信赖程度和对结果的接受程度。

\end{abstract}
